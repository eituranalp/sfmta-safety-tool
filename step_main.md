## What main.py Does

One job: expose the `scored_zones` PostgreSQL table as HTTP endpoints
that Orchestrate can call via OpenAPI tool integration.

Five endpoints total:
- GET /health         — keep-alive ping for UptimeRobot
- GET /query          — main flexible query endpoint
- POST /ingest        — trigger DataIngestionModule manually
- POST /score         — trigger RiskScoringModule manually
- GET /openapi.json   — auto-generated by FastAPI, do not write this

## The /query Endpoint

This is the core endpoint. All parameters are optional. FastAPI builds
the SQL query dynamically based on what Orchestrate passes.

### Parameters

```
question     string   optional  Karl's raw question (informational only,
                                not used for SQL — Orchestrate extracts
                                the structured params itself)
street_name  string   optional  Exact or partial street name to filter by
metric       string   optional  Column to sort by: "final_score",
                                "crash_count", "fatality_count",
                                "complaint_count", "recency_score"
limit        int      optional  Number of results to return (default 10,
                                max 50)
lat_min      float    optional  Bounding box minimum latitude
lat_max      float    optional  Bounding box maximum latitude
lng_min      float    optional  Bounding box minimum longitude
lng_max      float    optional  Bounding box maximum longitude
```

### Query Logic

Build the SQL query dynamically:

1. Start with base:
   SELECT location_name, crash_count, fatality_count, complaint_count,
          recency_score, final_score, latitude, longitude
   FROM scored_zones

2. If street_name provided:
   Add: WHERE location_name ILIKE '%{street_name}%'

3. If lat/lng bounding box provided (all four values present):
   Add: WHERE latitude BETWEEN lat_min AND lat_max
        AND longitude BETWEEN lng_min AND lng_max

4. If both street_name and bounding box provided:
   Use AND to combine both WHERE conditions

5. If metric provided and is a valid column name:
   Add: ORDER BY {metric} DESC
   Otherwise default: ORDER BY final_score DESC

6. Always add: LIMIT {limit} (default 10, max 50)

### Response Format

Always return:
```json
{
  "count": 3,
  "data": [
    {
      "location_name": "MISSION ST",
      "crash_count": 47,
      "fatality_count": 2,
      "complaint_count": 23,
      "recency_score": 0.82,
      "final_score": 87.3,
      "latitude": 37.758,
      "longitude": -122.418
    }
  ],
  "message": ""
}
```

When no parameters are passed return:
```json
{
  "count": 0,
  "data": [],
  "message": "No query parameters provided."
}
```

When scored_zones is empty return:
```json
{
  "count": 0,
  "data": [],
  "message": "No scored data available yet. Run /score first."
}
```

### Security: SQL Injection Prevention
NEVER use f-strings to inject column names directly into SQL without
validation. Validate metric against an explicit allowlist:

```python
VALID_METRICS = {
    "final_score", "crash_count", "fatality_count",
    "complaint_count", "recency_score"
}
if metric and metric not in VALID_METRICS:
    metric = "final_score"  # safe fallback
```

## APScheduler Setup

Add a daily scheduled job inside the FastAPI startup event:
- Run at 6:00 AM Pacific time daily
- Call ingest then score in sequence
- Log success or failure

```python
from apscheduler.schedulers.background import BackgroundScheduler

scheduler = BackgroundScheduler()

@app.on_event("startup")
def start_scheduler():
    scheduler.add_job(daily_pipeline, 'cron', hour=6, minute=0)
    scheduler.start()

    scheduler.shutdown()
```

## How to Verify It Works

### Test 1: Health check
```bash
curl http://localhost:8000/health
```
Expected: `{"status": "ok"}`

### Test 2: No parameters
```bash
curl http://localhost:8000/query
```
Expected: `{"count": 0, "data": [], "message": "No query parameters provided."}`

### Test 3: Top streets by score
```bash
curl "http://localhost:8000/query?metric=final_score&limit=5"
```
Expected: 5 rows from scored_zones ordered by final_score DESC
(Will return empty data message if scored_zones has no rows yet)

### Test 4: Street name search
```bash
curl "http://localhost:8000/query?street_name=MISSION&limit=3"
```
Expected: Rows where location_name contains "MISSION"

### Test 5: Bounding box
```bash
curl "http://localhost:8000/query?lat_min=37.748&lat_max=37.767&lng_min=-122.430&lng_max=-122.400&limit=10"
```
Expected: Rows within that lat/lng range

### Test 6: OpenAPI spec generated
Open browser: http://localhost:8000/openapi.json
Expected: Full OpenAPI spec JSON — this is what gets exported for Orchestrate

### Test 7: Interactive docs
Open browser: http://localhost:8000/docs
Expected: Swagger UI with all endpoints listed and testable
